Project Overview

把展厅里的低负担采集，转化成展后可回看、可分享的私人展册。

展迹不做通用相册，也不急着做展览社区，而是聚焦看展用户的真实限制：现场时间短、输入不方便、信息容易散落。产品将记录拆成两个阶段，先采集，再整理。

Problem

看展用户不是不想记录，而是记录动作经常打断观看本身。

现场输入成本高

展厅里有动线、光线和社交限制，用户不适合长时间打字整理。

信息分散难归档

照片、票根、标签、地点和感受常散落在相册、备忘录和社交平台。

分享整理成本高

想发看展内容时，还要重新选图、排版、补充文字和作品信息。

Solution

用“现场轻记录 + 展后整理 + 分享生成”降低记录阻力。

产品把看展记录拆成两个阶段：现场只做必要采集，展后再补全作品信息、整理感受，并生成适合社交平台的分享内容。

01

进入展册

查看正在展中的记录和最近展记，明确沉淀位置。

02

开始看展

新建展迹，填入展名、日期、地点、天气和心情。

03

现场采集

以拍展品为主，支持补拍标签、票根和一句感受。

04

标签补录

作品标签进入待确认，不要求用户现场完成校对。

05

生成分享

按长图、九宫格或封面卡导出，完成从私人记录到分享表达的转化。

Interface

界面围绕“看展前、看展中、看展后”三个时段组织。

移动端设计重点是降低现场操作负担，让主动作足够明确，同时把展后整理和分享变成自然延伸。

展后整理 / 分享生成

Product Thinking

产品思考重点在于尊重线下场景的限制。

展迹的核心不是功能堆叠，而是把线下观展的节奏转译成移动端低打扰交互。

产品层

  • 把记录拆成现场采集和展后整理两个阶段。
  • 让拍展品成为主动作，标签和感受作为辅助动作。
  • 默认服务私人沉淀，再提供分享生成能力。

体验层

  • 减少现场输入，避免打断观看。
  • 用待确认机制承接 OCR 或标签补录的不确定性。
  • 用分享模板降低用户二次排版成本。
