# 家plus：PRD

## 1. 文档信息

- 产品名称：家plus
- 产品形态：iOS App Demo
- 当前版本：MVP v0.1
- 目标用途：作品集展示、比赛 Demo、AI 产品经理求职项目

## 2. 产品目标

帮助精装修新房业主通过拍摄房间照片，快速获得基于真实空间的软装改造效果图、家具推荐清单和分步落地建议。

MVP 需要跑通完整核心链路：

```text
拍照 / 上传照片
-> 选择空间、风格、预算和家庭需求
-> AI 生成改造方案
-> 查看效果图、家具推荐和执行清单
-> 保存方案
```

## 3. 目标用户

核心用户是精装修交付的新房业主，尤其是刚收房、准备入住或正在购买家具软装的人群。

次级用户包括二手房轻改造用户和租房软装改造用户，但 MVP 不围绕这两类用户展开复杂能力。

## 4. 使用场景

### 场景一：新房刚交付

用户拿到精装房后，觉得空间缺少个性，希望快速看到不同软装风格下的改造效果。

### 场景二：准备购买家具

用户准备购买沙发、茶几、电视柜、床、窗帘等软装大件，但担心颜色、尺寸和风格不协调。

### 场景三：预算有限

用户希望分阶段改造，想知道哪些家具或软装应该优先购买。

## 5. MVP 功能范围

### 包含功能

- 首页入口
- 场景选择
- 拍照或上传照片
- 空间类型选择
- 风格选择
- 预算选择
- 家庭需求选择
- AI 生成状态展示
- 改造前后效果展示
- 空间问题诊断
- 家具推荐清单
- 家具推荐理由
- 分步改造建议
- 方案保存

### 不包含功能

- 登录注册
- 真实支付
- 真实电商购买
- 真实施工报价
- 设计师咨询
- AR 量房
- 3D 建模
- 社区分享
- App Store 上架

## 6. 页面结构

### 6.1 首页

目标：让用户理解产品价值，并进入核心流程。

核心内容：

- 产品主文案：拍一张精装房照片，生成你的软装改造方案
- 副文案：AI 帮你预览效果图、推荐家具、规划购买顺序
- 主按钮：开始改造
- 次入口：查看示例方案

交互：

- 点击“开始改造”进入场景选择页。
- 点击“查看示例方案”进入预置案例结果页。

### 6.2 场景选择页

目标：确认用户改造类型。

选项：

- 精装房软装
- 二手房轻改
- 租房改造

MVP 默认主推“精装房软装”。其他选项可以进入同一流程，但结果页文案保持轻量，不扩展硬装能力。

交互：

- 用户选择场景后进入上传页。

### 6.3 拍照 / 上传页

目标：获取用户真实空间照片。

核心内容：

- 上传照片按钮
- 拍照按钮
- 示例图提示
- 隐私说明：照片仅用于生成当前改造方案

图片要求：

- 推荐横向或竖向完整拍摄房间一角。
- 尽量包含地面、墙面、窗户和主要家具区域。
- 避免过暗、过曝、严重遮挡。

交互：

- 上传成功后显示图片预览。
- 用户确认后进入偏好选择页。

### 6.4 偏好选择页

目标：收集生成方案所需的结构化信息。

输入项：

- 空间类型：客厅、卧室、餐厅、书房
- 风格偏好：现代简约、原木风、奶油风、轻法式、复古风
- 预算档位：轻改造 3000-8000 元、中等改造 8000-20000 元、品质改造 20000 元以上
- 家庭需求：有孩子、养宠物、重视收纳、喜欢大沙发、需要办公区

交互：

- 空间类型、风格、预算为必选。
- 家庭需求为可选多选。
- 点击“生成方案”进入 AI 生成页。

### 6.5 AI 生成页

目标：降低等待焦虑，展示 AI 处理过程。

展示步骤：

- 正在理解空间结构
- 正在匹配软装风格
- 正在生成改造效果图
- 正在推荐家具清单
- 正在整理改造步骤

交互：

- 生成完成后自动进入结果页。
- 如果生成失败，显示重试按钮和返回修改偏好的入口。

### 6.6 结果页

目标：展示完整方案，让用户看到视觉结果和执行路径。

模块：

- 原图 / 改造图对比
- 空间概况
- 当前空间问题诊断
- 风格方案说明
- 推荐家具清单
- 推荐理由
- 优先购买清单
- 可延后购买清单
- 分步改造建议
- 保存方案按钮

交互：

- 支持切换原图和改造图。
- 点击家具卡片查看详情。
- 点击保存方案进入保存成功页。

### 6.7 家具详情页

目标：解释单件家具为什么被推荐。

内容：

- 商品名称
- 品类
- 风格标签
- 价格区间
- 颜色
- 材质
- 推荐理由
- 适合放置位置

交互：

- 返回结果页。
- 加入方案清单。

### 6.8 方案保存页

目标：完成产品闭环。

内容：

- 保存成功提示
- 方案名称
- 生成时间
- 再生成一个方案
- 返回首页

MVP 中方案保存可以使用本地状态或静态模拟，不要求真实账号体系。

## 8. 优先级

### P0

- 上传照片
- 选择空间、风格、预算
- 生成或模拟改造效果图
- 展示家具推荐
- 展示改造步骤

### P1

- 家庭需求选择
- 原图 / 改造图对比
- 家具详情页
- 方案保存

### P2

- 多方案对比
- 方案分享
- 风格收藏
- 真实商品跳转

## 9. 异常和边界

- 图片过暗：提示用户重新上传更清晰照片。
- 图片不是室内空间：提示当前照片无法识别房间。
- 用户未选择必填项：按钮置灰或提示补全。
- AI 生成失败：允许重试或查看示例方案。
- 推荐商品不足：展示通用品类建议，不强行编造商品。

## 10. 验收标准

- 用户可以完整完成一次核心流程。
- 结果页同时包含效果图、问题诊断、家具推荐和执行步骤。
- 家具推荐理由具体，不是泛泛的风格描述。
- 预算档位能影响推荐内容。
- 页面流程适合录制 1-2 分钟 Demo 视频。
