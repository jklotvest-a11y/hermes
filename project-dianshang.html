Project Overview

把“AI 生图”拆成设计师能控制、运营能表达、团队能复用的生产流程。

项目聚焦内衣电商场景，不追求通用生图工具的大而全，而是围绕商品真实性、品牌调性、平台合规和素材入库做 MVP 产品设计。

PRD

电商生图 AI 中台 PRD 文档

完整 PRD 已嵌入当前页面，方便面试官直接阅读产品背景、MVP 范围、功能设计和验收标准。

Problem

内衣电商出图不是简单“换背景”，更难的是效率、真实、合规同时成立。

重复出图消耗设计时间

主图、详情页、卖点图和多尺寸素材高频重复，创意价值有限但占用大量工时。

需求表达不结构化

运营通过聊天或文档描述商品信息，设计师需要反复确认标题、卖点、尺寸和参考图。

类目合规风险更高

内衣类目需要控制版型真实性、露肤尺度、身体表达和平台审核风险。

Solution

用对话式 Agent 串起从需求输入到候选图入库的 MVP 闭环。

系统先把自然语言需求解析为结构化任务，再结合商品图、参考图、模板和类目规则生成候选图，最后由设计师筛选、下载或入库。

1

上传资料

上传商品图、参考图和补充规则。

2

任务解析

识别图片类型、尺寸、模板和卖点。

3

类目校验

检查商品真实性、表达尺度和平台风险。

4

候选生成

输出多张候选图供人工筛选。

5

下载入库

保留可用素材，并沉淀任务历史。

Prototype

可交互低保真原型：视觉智能体工作台。

原型包含视觉智能体、模板库管理、提示词反推、视频智能生成和任务历史，可直接点击左侧模块切换。

Product Thinking

MVP 保持收敛：先验证基础出图效率，再扩展到资产管理和模型能力。

范围取舍

不做复杂权限、在线精修画布和商业化计费，先完成对话式生图、候选筛选、下载与入库。

类目规则

围绕版型、颜色、面料纹理、露肤尺度和卖点表达建立可用/不可用标准。

指标设计

以单个商品基础出图耗时为北极星指标，辅以可用率、下载率、入库率和重新生成率。
